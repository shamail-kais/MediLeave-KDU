# MediLeave – Medical Leave Management Mobile Application

## Overview

MediLeave is a mobile application developed to simplify and digitize the medical leave application and approval process. This project was created as part of a 3rd Year Final Project. The system allows users to submit medical leave requests, upload supporting medical documents, and track the approval status through a mobile application.

Administrators can review requests, verify documents, and approve or reject leave applications through a centralized backend system.

## Technology Stack

**Frontend**

* Flutter

**Backend**

* Spring Boot
* REST API

**Database**

* PostgreSQL

**Version Control**

* Git & GitHub

## Features

* User registration and login
* Submit medical leave requests
* Upload medical certificates
* Track leave status
* Admin approval and rejection system
* Secure REST API communication

## System Architecture

The application follows a client–server architecture:

Mobile App (Flutter) → REST API (Spring Boot) → Database (PostgreSQL)

The Flutter mobile application communicates with the Spring Boot backend through REST APIs, while PostgreSQL manages all persistent data such as user accounts, leave requests, and approval records.

## Project Objective

The objective of this project is to develop a mobile-based medical leave management system that reduces paperwork and improves efficiency in handling leave requests.
